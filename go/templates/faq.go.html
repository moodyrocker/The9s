Basics

What is a HTTP status code?

HTTP status codes are standard response codes given by website servers on the internet. The codes help identify the cause of the problem when a web page or other resource does not load properly. To keep things simple 2XX and 3XX are good responses 4XX and 5XX are bad, for a full list of code follow the link.

Technical details

How do you get this data?

We have a number instances running in AWS in 6 global locations, each instance makes a request to a URL that emulates a real user. In the sense that a browser is opened and a request is made the information that we get back is recorded in a central database.

What you see in the graph is an aggregation of the response times, in future versions of TheNINEs we will enable a break down of times from each location. This will allow you to see where the customer is seeing slowness.

https://www.sitespeed.io/

How often do you check websites?

In the Alpha version of the9s, we test each URL every 5mins. In future versions, we hope to test as frequent as 10s. From experience, this is not that useful however this is more about showing capabilities.
